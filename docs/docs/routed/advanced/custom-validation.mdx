# Custom Validation Rules

While Routed provides many built-in validation rules, you can also create custom rules for your specific needs.

```dart title="Import validation helpers"
import 'package:routed/validation.dart';
```

## Creating a Custom Rule

To create a custom validation rule, implement the `ValidationRule` interface:

```dart
class CustomRule implements ValidationRule {
  @override
  String get name => 'custom_rule_name';

  @override
  String message(dynamic value, [List<String>? options]) =>
      'Custom error message';

  @override
  bool validate(dynamic value, [List<String>? options]) {
    // Implement validation logic
    return true; // or false based on validation
  }
}
```

## Example Rules

### Password Strength Rule

```dart
class PasswordStrengthRule implements ValidationRule {
  @override
  String get name => 'strong_password';

  @override
  String message(dynamic value, [List<String>? options]) =>
      'Password must contain at least one uppercase letter, one lowercase letter, one number, and one special character';

  @override
  bool validate(dynamic value, [List<String>? options]) {
    if (value == null || value is! String) return false;
    
    return RegExp(
      r'^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$'
    ).hasMatch(value);
  }
}
```

### Credit Card Rule

```dart
class CreditCardRule implements ValidationRule {
  @override
  String get name => 'credit_card';

  @override
  String message(dynamic value, [List<String>? options]) =>
      'Must be a valid credit card number';

  @override
  bool validate(dynamic value, [List<String>? options]) {
    if (value == null) return false;
    
    // Remove spaces and dashes
    final number = value.toString().replaceAll(RegExp(r'[\s-]'), '');
    
    // Check if contains only digits
    if (!RegExp(r'^\d+$').hasMatch(number)) return false;
    
    // Implement Luhn algorithm
    int sum = 0;
    bool alternate = false;
    
    for (int i = number.length - 1; i >= 0; i--) {
      int n = int.parse(number[i]);
      if (alternate) {
        n *= 2;
        if (n > 9) n = (n % 10) + 1;
      }
      sum += n;
      alternate = !alternate;
    }
    
    return sum % 10 == 0;
  }
}
```

### Phone Number Rule

```dart
class PhoneNumberRule implements ValidationRule {
  @override
  String get name => 'phone';

  @override
  String message(dynamic value, [List<String]? options]) =>
      'Must be a valid phone number';

  @override
  bool validate(dynamic value, [List<String]? options]) {
    if (value == null) return false;
    
    // Basic US phone number format
    return RegExp(r'^\d{3}-\d{3}-\d{4}$').hasMatch(value.toString());
  }
}
```

## Using Custom Rules

### Registration

First, register your custom rules (pass a factory; rule names must be unique):

> Note: `ValidationRuleRegistry` is available by default in bare mode (`Engine()`) and is also registered by `RoutingServiceProvider` in full mode. If you manually remove it, re-register one before calling `engine.container.get<ValidationRuleRegistry>()`.

```dart
void main() {
  final engine = Engine();
  final rules = engine.container.get<ValidationRuleRegistry>();
  rules.register(() => PasswordStrengthRule());
  rules.register(() => CreditCardRule());
  rules.register(() => PhoneNumberRule());
  // ... rest of setup
}
```

### Usage in Validation

```dart
router.post('/register', (ctx) async {
  await ctx.validate({
    'username': 'required|string|min:3',
    'password': 'required|strong_password',
    'phone': 'required|phone',
    'credit_card': 'required|credit_card'
  });
  
  // Process valid registration...
});
```

## Rules with Options

You can create rules that accept options for more flexible validation:

```dart
class EnumRule implements ValidationRule {
  @override
  String get name => 'enum';

  @override
  String message(dynamic value, [List<String>? options]) =>
      'Value must be one of the allowed values';

  @override
  bool validate(dynamic value, [List<String>? options]) {
    if (value == null || options == null) return false;
    return options.contains(value.toString());
  }
}

// Usage:
await ctx.validate({
  'status': 'required|enum:pending,active,completed'
});
```
